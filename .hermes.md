# Hermis Life OS Workspace Instructions

This directory is the user's Life OS workspace.

## Directory roles

- raw/captures: immutable messy user captures
- raw/transcripts: chat or meeting transcripts
- raw/documents: source documents
- raw/web-clips: saved web source material
- memory/ledger: append-only timeline of source-linked life facts
- memory/review: proposed durable memories awaiting review
- memory/curated: accepted durable memories
- wiki: LLM-maintained compiled understanding
- state: current tasks, reminders, commitments, habits, research queue
- reports/morning: morning reports
- reports/nightly: nightly processing output
- reports/work: work parsing review output
- reports/weekly: weekly reviews
- data/finance: structured finance tracker logs and local-only daily finance records
- data/work: structured confirmed work logs and local-only work capture/item records
- research/watchlists: topics to monitor
- research/nightly: overnight research summaries
- scripts: deterministic helper scripts
- manifests: source and wiki change tracking

## Non-negotiable rules

1. Never modify raw captures except to append new captures.
2. Never delete raw files.
3. Do not put raw dumps into Hermes built-in memory.
4. Keep the wiki concise, sourced, and updated.
5. When a claim changes, supersede the old claim and update contradictions/changelog.
6. Use OpenViking for recall and retrieval, but keep the filesystem as source truth.
7. Write all morning and nightly reports to files, even if also delivered by gateway.
8. Prefer one clear next action over a long generic plan.
9. External actions require explicit approval.
10. If something is uncertain, create an open question.
11. Index only wiki/ and user-approved memory/curated/ into OpenViking by default.
12. Do not index raw captures, raw documents, tracker logs, databases, env files, or backups unless the user explicitly approves that exact indexing action.
13. For finance, answer from wiki/domains/money.md and compact daily/nightly summaries first; inspect raw finance logs or DB rows only for audit/correction.
14. Never index raw finance messages, tracker DB rows, or detailed finance logs into OpenViking by default.
15. Normal finance rollups belong in weekly reports. Daily reports mention money only for commitments, promises to pay, or deadlines.
16. Finance Discord input should be processed automatically by the AI-led finance review processor; scripts may transport, validate, and apply structured JSON but must not infer amounts/categories from raw human finance text.
17. Manual money commands are fallback only.
18. `memory/curated` may contain user-approved memories or safe nightly auto-promoted memories. Safe auto-promotion requires source-backed, high-confidence, non-sensitive, non-contradictory durable facts.
19. Normal work Discord input is review-first: save raw capture and draft parse, then confirm/correct/split/ignore/clarify through AI-led work review before creating `work_items`.
20. Work review scripts may transport, validate, and apply structured JSON but must not treat draft parses as final task truth.

## Capture classification

Classify every capture into one or more:
- task
- reminder
- commitment
- habit log
- durable fact
- idea
- research topic
- emotional signal
- needs-answer
- finance

## Wiki maintenance

Every important wiki update should include:
- source path
- source date
- confidence: low / medium / high
- status: draft / active / superseded
- last updated date
